What is Mapper?

Understanding of mapper from the ground up.

September 28, 2017 Nathaniel Saul

3 minute read

As one of the main tools from the field of Topological Data Analysis, mapper has been shown to be particularly useful for exploring high dimensional point cloud data.This post will walk through the mapper construction from an intuitive perspective and demonstrate its use on a toy example.

Motivation

Mapper is way to construct a graph (or simplicial complex) from data in a way that reveals the some of the topological features of the space. Though not exact, mapper is able to estimate important connectivity aspects of the underlying space of the data so that it can be explored in a visual format. This is an unsupervised method of generating a visual representation of the data that can often reveal new insights of the data that other methods cannot.

Mapper

Mapper can be viewed from two perspectives, one very informal and one very mathematical. The formal definition of mapper is a simplicial complex constructed by taking the nerve of the pullback of an overlapping cover of a lens. This definition boils down to simply splitting up the data points into buckets, clustering the points within each bucket, and then stringing the clusters together into a simplicial complex.

To construct the mapper, we need to define two pieces

  • the lens
  • the cover

The lens is a function that we look at the data through. It is a summarization of your data in a much lower dimensional space. If you are looking at 3D images, the lens function might be the height, or y-axis of the data. If your data is extremely high dimensional, your lens might TSNE or another dimensionality reduction.

Example

In these images, we have noisy data sampled from a circle. As a filter function, we have have chosen the y-axis, or height of the data points. The y-axis is then split into 10 sequential bins, each with some overlap with neighboring bins.

We then find all points within each bin, and with respect to their position in the original space, we run any clustering algorithm. The second image shows the resulting clusters. Notice that because the original cover was overlapping, some of these clusters have data points in common.

We then compute the nerve of these clusters. For each cluster, we add a vertex, and between any two clusters that share points in common, we add an edge between their respective vertices.

The result is a graph (or simplicial complex)that captures the structure of the original data with respect to our lens. From the noisy data that was obviously drawn from a circle, we constructed a graph that is one big loop.

From a homological perspective, the extra vertex and edge at the bottom is not very meaningful. In fact, if we decreased the number of bins used, these features would disappear while the loop would remain.

To explore an mapper in practice, checkout KeplerMapper.
